    Describe the bug

    Im experiencing an issue, where the TypeScript compiler fails to transform JavaScript decorators to the configured target environment ("es2019"). The exact error Im receiving is "Transforming JavaScript decorators to the configured target environment ("es2019") is not supported yet." This issue has only started recently, and previously the project was functioning correctly.

    The project is configured to use TypeScript with experimental decorators enabled. Despite setting up the tsconfig.json correctly, Im encountering a transformation error when the TypeScript compiler tries to handle JavaScript decorators for the "es2019" environment.

    Additional context

    The following configuration in nuxt.config.ts also doesnt work and begins to terminate the build with an error: RollupError: Unexpected character '@' (Note that you need plugins to import files that are not JavaScript).

    export default defineNuxtConfig({
  nitro: {
    esbuild: {
      options: {
        target: 'esnext'
      }
    },
    typescript: {
      tsConfig: {
        compilerOptions: {
          experimentalDecorators: true
        }
      }
    }
  }
})
